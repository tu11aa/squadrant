Bucket 2 · Cockpit-wide hardening · filed issues

Each item is a tracked GitHub issue. Bodies cite orca prior art with file:line and name the cockpit subsystem to touch.

#89watchdog: key heartbeats by dispatch-id, not task-id (sling pattern, part 1) enhancement

Why. Cockpit's heartbeat is task-keyed; a late heartbeat from a dead retry can refresh the timer of a hung live one.
Orca prior art. coordinator.ts:274–301 — heartbeats keyed by dispatch context, not task.
Cockpit change. Sub-record attempts on TaskRecord (src/control/types.ts + store.ts); watchdog reads attempt-level heartbeats. Pure reducer preserved.
Acceptance. Replaying an old attempt's heartbeat cannot mask a hung new dispatch; integration test added.

#90watchdog: warn-don't-autofail on stale; surface to Captain (default policy) enhancement

Why. Killing a slow-but-correct worker costs more than letting a hung one hold a slot until a human notices.
Orca prior art. coordinator.ts:227–241 — warnStaleDispatches deliberately only warns.
Cockpit change. Default policy = warn + surface to Captain (control-plane event + reactor notification); stalled stays non-terminal; hard-kill behind a flag.
Acceptance. Stalled task remains non-terminal; Captain receives a structured "needs attention" event; per-project hard-kill budget configurable.

#91control-plane: model dispatch attempts as sub-records (sling pattern, part 2) enhancement

Why. Retries currently dirty task state; the 3rd-attempt retry can mis-read the 1st attempt's error as current state. Auditing is muddy.
Orca prior art. orca's "sling pattern" — dispatch contexts are separate rows; circuit-breaker (3 failures → circuit_broken) lives on the context (coordinator.ts:562–568).
Cockpit change. DispatchAttempt sub-record with attemptId, startedAt, pid, error, exitCode, lastHeartbeat, circuitBroken, resumeRef; TaskRecord.attempts; reducer/store/watchdog operate on attempts.
Depends on. #86 (resumeRef lives on the attempt). Pairs with #89.

#92cockpitd: add a wire PROTOCOL_VERSION constant and handshake enhancement

Why. No on-the-wire version means a future wire-shape change silently breaks in-flight clients during a rolling restart.
Orca prior art. daemon/types.ts:6 — PROTOCOL_VERSION = 7, PREVIOUS = [1..6]. Daemons survive app updates.
Cockpit change. Constant in src/control/protocol.ts; server includes it in replies (or a hello frame); client refuses mismatched versions with a clear error.
Related. Prerequisite for #87 (protocol schema validation).

#93cockpitd: coalesce concurrent restarts (in-process Promise + filesystem lock) enhancement

Why. ensureDaemon() runs on every CLI invocation; the plist-diff mitigation closes most of the window but a race remains.
Orca prior art. daemon/daemon-init.ts:40–46 — restartInFlight promise so two restart triggers can't both enter teardown.
Cockpit change. In src/control/launchd.ts, in-process Promise gate + a daemon-side single-instance lock at ~/.config/cockpit/daemon.lock (flock / O_EXCL) so concurrent CLI processes serialize.
Acceptance. Two simultaneous cockpit invocations during a restart cannot both bootout/bootstrap; one wins, others wait.

#94protocol: keepalive framing on AF_UNIX so long dispatches don't trip idle timers enhancement

Why. A future interactive subscribe stream (or a slow dispatch confirmation) can trip socket idle timers with no traffic.
Orca prior art. runtime-rpc.ts — {"_keepalive":true} every 10s past 10s idle; clients discard.
Cockpit change. Same frame in src/control/protocol.ts; all consumers MUST discard. Pairs with #87 (becomes a known frame, not invalid).
Related. #87. Prerequisite for the interactive-codex streaming subscribe channel.

#95dispatch: pre-flight worktree git-drift check (skip-not-fail on stale base) enhancement

Why. A crew dispatched into a worktree whose base has drifted may waste a full dispatch reasoning against stale ground.
Orca prior art. coordinator.ts:481–521 — drift check before dispatch; >20 commits behind base = skip, don't fail (burning the breaker would convert recoverable → hard-fail).
Cockpit change. In src/commands/crew-control.ts dispatch path, compute drift when cwd is a worktree; over threshold (default 20) return a structured stale-base deferral the Captain or decision-gate (Bucket 1 item 4) resolves.
Acceptance. Dispatch into stale worktree doesn't burn a crew/turn; returns a non-failure deferral.

Bucket 3 · Multi-provider expansion patterns (architecture notes; no issues today)

  • TUI_AGENT_CONFIG-style single-source-of-truth row per provider. Orca onboards a new agent with one row + (optional) hook-service. Cockpit's runtime-driver slot should grow a similar per-provider record (detect/launch/prompt-injection-mode/preflight-trust/foot-guns). Consolidate when the 4th interactive provider lands.
  • Per-provider foot-gun catalogue baked into the driver. TUI-killing non-interactive flags (codex --ephemeral, copilot --prompt); argv > Win/SSH cmdline limits → stdin; node-pty NAPI dispose-before-kill (if cockpit ever pty-hosts); treat resumeRef as opaque (orca #1148).
  • Codex 0.129+ silent hook-drop / trust-hash precondition. Encoded in Bucket-1 item 2. General lesson: codex silently degrades when config preconditions unmet — never treat silence as success.

Bucket 5 · Future-architecture notes (reference for later)

  • Lightweight remote-relay pattern — SCP'd daemon over SSH + framed JSON-RPC + grace-period PTY survival on a unix socket across disconnects, hosting the same hook pipeline remotely (orca src/relay/). Pattern of record for when "cockpit goes remote" is a goal.
  • WS + E2EE + pairing + device-registry (orca's mobile companion). Not cockpit's direction today (Obsidian hub-spoke covers reporting). Recorded for completeness.

Where cockpit is genuinely ahead — do NOT regress

These are not action items — they are guardrails. Orca lacks them; future cockpit work must not weaken them.

  1. Pure reduce(state, event) + typed per-task JSON + TERMINAL_STATES. Orca's live-agent truth is an opaque xterm blob; cockpit's auditable typed state is strictly better.
  2. Anti-false-done invariant as a stated principle. Orca has the bug class (#1437 stuck-spinner, tui-idle null→idle hangs) and patches reactively. Cockpit elevates it to an invariant — do not weaken.
  3. Non-invasive footprint (~/.config/cockpit). Orca writes into 8 third-party agent home dirs. If cockpit ever adopts agent-hooks, copy orca's mitigations (filename-scoped matchers + hash-matched cleanup) — not the blast radius.
  4. Self-improving subsystems (learnings, LLM-wiki, self-heal-on-invoke). Zero orca analog. Keep investing.
  5. Orchestration as an LLM session, not hardcoded code. Orca's coordinator can't AI-decompose (deferred); Command/Captain already are that phase.

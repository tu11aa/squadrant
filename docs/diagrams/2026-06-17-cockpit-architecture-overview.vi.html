1 · 5 tầng của cockpit

Cockpit là lớp điều phối đa-agent. Đọc từ trên xuống: người dùng gõ CLI → CLI spawn các role-session + nói chuyện với control plane (daemon) → daemon dùng các adapter để điều khiển agent & terminal → mọi thứ ghi xuống persistence.

③ control plane = nơi daemon & cockpitd sống — giữa roles (trên) và adapters (dưới)

2 · Một lệnh đi xuyên kiến trúc

Ví dụ cockpit crew spawn …: CLI → daemon (ghi ledger) → adapter cmux (tạo surface, spawn agent) → crew chạy → khi xong, daemon → mailbox → delivery → captain. daemon/cockpitd là trạm trung chuyển ở giữa.

Chi tiết khúc "done → mailbox → captain": xem deck 2026-06-16-daemon-architecture-and-flow.html.

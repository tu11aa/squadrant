⚓ Cockpit — Monorepo Architecture

6-package layout · post-reorg 2026-06-18
space / → · A = autoplay

1 · Package DAG

Six internal packages in a strict one-way dependency graph. @cockpit/shared is the leaf — no internal deps. @cockpit/cli is the root — depends on everything. Nothing flows backwards.

shared core agents workspaces web cli

2 · Driver Seams

Two plugin slots isolate cockpitd from concrete tool choices. Swap a driver without touching core. The agent seam abstracts which AI runs; the workspace seam abstracts how the terminal, vault, and notifier work.

3 · Build Artifacts & Runtime Model

pnpm build (tsup) produces two self-contained bundles — all internal packages are inlined. The Captain is an interactive agent session in cmux; it spawns Crew members via cockpit crew spawn.

4 · Package Reference

Each package ships its own tsconfig.json + src/. All are private (not published to npm). @cockpit/cli is the only entry point for the outside world.
